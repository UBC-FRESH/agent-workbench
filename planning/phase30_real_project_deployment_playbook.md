# Phase 30 Real-Project Deployment Playbook

Phase 30 packages the P26-P29 workflow into a reusable playbook for real
UBC-FRESH projects.

## Included Workflow Pieces

- P26 cross-project eval support;
- P27 isolated proposal packs and supervisor decision packets;
- P28 claim-review fields and checklist; and
- P29 model/repeat comparison reports for existing eval summaries.

## Deployment Boundary

The playbook keeps Agent Workbench as a supervisor-side tool. Workers may help
with proposal generation and bounded evidence review, but tracked-file mutation,
GitHub mutation, release work, and closeout remain supervisor responsibilities.

## Decision

The workflow is ready for cautious real-project proposal-assist use. It is not
ready for autonomous implementation or delegated GitHub closeout.
